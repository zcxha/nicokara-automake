# Nicokara 项目中文使用说明

## 1. 适用场景

当你有：

- 一段歌曲视频 `mp4`
- 一份正确歌词 `txt`

并希望得到：

- 自动对齐歌词
- KTV 逐词高亮字幕
- 汉字上方假名标注
- 最终ニコカラ成品视频

就可以使用这个项目。

## 2. 环境准备

### 2.1 系统依赖

请先确保系统里有：

- `ffmpeg`
- `ffprobe`

### 2.2 推荐安装命令

```bash
uv tool install --with packaging --with setuptools whisper-timestamped
uv tool install --with torchcodec demucs
uv pip install --python .venv/bin/python -e .
```

说明：

- `whisper-timestamped` 用来做词级歌词识别
- `demucs` 用来做人声分离
- 默认 Python 依赖里已经包含 `fugashi[unidic-lite]`、`pykakasi` 和 `Pillow`
- 默认假名链路会优先使用 `fugashi + JmdictFurigana / JmnedictFurigana`

如果你的机器不打算使用 GPU，Whisper 也推荐直接装 CPU 版工具环境：

```bash
uv tool install --force --torch-backend cpu --with packaging --with setuptools whisper-timestamped
```

如果你之前已经装过 `demucs`，建议补一次：

```bash
uv tool install --force --with torchcodec demucs
```

这样可以避免新版 `torchaudio` 在保存分离结果时因为缺少 `torchcodec` 而报错。

如果你的机器没有可用 CUDA，或者 CUDA 驱动版本太旧，更推荐直接安装 CPU 版工具环境：

```bash
uv tool install --force --torch-backend cpu --with torchcodec demucs
```

这样可以避免 `torchcodec` 在加载时继续去找 CUDA 相关动态库。

如果你还想显式启用 Sudachi 后端，可以额外安装：

```bash
uv pip install --python .venv/bin/python -e ".[sudachi]"
```

### 2.3 不建议直接用系统 `pip3`

如果你看到类似 `externally-managed-environment` 的报错，不要继续对系统 Python 使用 `pip3 install`。

这个项目更推荐：

- 用 `uv tool install ...` 安装 CLI 工具
- 用项目自己的虚拟环境安装 Python 依赖

## 3. 输入文件要求

### 3.1 视频

- 格式建议为 `mp4`
- 视频里应包含原始音轨

### 3.2 歌词

- 纯文本 `txt`
- 每行对应一句歌词
- 尽量使用最终想显示在字幕里的文本
- 建议去掉多余空行

## 4. 最常用命令

### 4.1 一条命令直接生成成片

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt"
```

默认会在当前目录生成一个 `input.nicokara/` 文件夹。

## 5. 输出目录里会有什么

假设输入视频是 `input.mp4`，会生成：

- `input.audio.wav`
  从视频中提取出来的单声道 WAV
- `input.vocals.wav`
  人声分离后的音频
- `input.vocals.wav.words.json`
  Whisper 词级时间戳结果
- `input.nicokara.json`
  正确歌词和 ASR 的对齐结果
- `input.nicokara.srt`
  行级字幕
- `input.karaoke.ass`
  带逐词高亮和假名标注的 KTV 字幕
- `input.nicokara.mp4`
  最终成片

## 6. 常用参数

### 6.1 指定输出目录

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --output-dir "build/my_song"
```

### 6.2 指定最终视频输出路径

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --output-video "release/my_song.nicokara.mp4"
```

### 6.3 指定 Whisper 模型

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --whisper-model medium
```

如果机器性能一般，先用默认的 `small` 即可。

### 6.4 指定设备

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --whisper-device cpu
```

如果你有可用 CUDA 环境，也可以改成：

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --whisper-device cuda
```

### 6.5 开启 Whisper VAD

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --whisper-vad
```

这个选项在某些噪声场景下有帮助，但不是所有歌曲都一定更好。

### 6.6 强制重跑

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --force
```

适合你修改了歌词或参数，希望覆盖旧结果时使用。

### 6.7 只生成字幕和 JSON，不烧录视频

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --skip-burn
```

适合你先想检查对齐结果和 ASS 样式时使用。

### 6.8 指定假名后端

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --reading-backend auto
```

可选值：

- `auto`
- `fugashi`
- `sudachi`
- `pykakasi`

默认推荐保留 `auto`。

### 6.9 指定 Sudachi 切分模式

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --reading-backend sudachi \
  --reading-split-mode C
```

### 6.10 指定外部 furigana 资源

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --furigana-resource "/path/to/JmdictFurigana.json.gz"
```

### 6.11 指定人工读音覆写

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --reading-overrides "ruby_overrides.json"
```

## 7. 复用已有中间结果

### 7.1 已经有人声音频

如果你已经有 `vocals.wav`，可以跳过 Demucs：

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --vocals "song.vocals.wav"
```

### 7.2 已经有 Whisper JSON

如果你已经有 `*.words.json`，可以直接跳过分离和识别：

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --asr-json "song.vocals.wav.words.json"
```

这是最快的调试方式，适合反复微调歌词或字幕样式。

## 8. 只想反复调对齐和字幕

当前仓库没有单独的“只对齐” CLI，最方便的调试方式是复用已有 ASR JSON，并加上 `--skip-burn`：

```bash
python3 build_nicokara_video.py "input.mp4" "lyrics.txt" \
  --asr-json "song.vocals.wav.words.json" \
  --skip-burn
```

这样会重新生成：

- `*.nicokara.json`
- `*.nicokara.srt`
- `*.karaoke.ass`

但不会重新烧录 MP4。

## 9. 歌词书写建议

为了得到更好的效果，建议：

- 一行就是一句
- 文本尽量和想显示在画面上的歌词完全一致
- 不要把多个段落挤在同一行
- 如果你想手动指定高亮边界，可以在适当位置加空格

如果不加空格也没关系，项目会自动把日文切成更细的高亮单元。

## 10. Ruby 诊断工具

生成 `*.nicokara.json` 后，可以用下面的命令筛出值得人工复核的汉字词：

```bash
nicokara-ruby-diagnostics song.nicokara.json > suspicious_ruby.json
```

这个工具默认会标出：

- `ruby_source == "backend"` 的词
- `ruby_text` 缺失的词
- `coverage` 低于阈值的词

也可以顺手生成一个人工覆写模板：

```bash
nicokara-ruby-diagnostics \
  song.nicokara.json \
  --overrides-template ruby_overrides.json
```

## 11. 常见问题

### 11.1 报错找不到 `whisper_timestamped`

执行：

```bash
uv tool install --with packaging --with setuptools whisper-timestamped
```

### 11.2 报错找不到 `demucs`

执行：

```bash
uv tool install --force --with torchcodec demucs
```

### 11.3 报错找不到 `ffmpeg` 或 `ffprobe`

说明系统层面的媒体工具还没安装，需要先安装 FFmpeg。

### 11.4 输出字幕对齐不够理想

先检查：

- 歌词文本是否正确
- 歌词是否一行一句
- 原视频音质是否太差
- 是否需要换更大的 Whisper 模型
- 是否需要补 `--reading-overrides`

推荐的排查顺序：

1. 先用 `--skip-burn` 生成中间文件
2. 检查 `*.nicokara.json`
3. 检查 `*.karaoke.ass`
4. 运行 `nicokara-ruby-diagnostics *.nicokara.json`
5. 再决定是否要改歌词、改模型、改覆写或改样式

### 11.5 首次运行很慢

这是正常的，因为：

- Demucs 可能需要下载模型
- Whisper 可能需要下载模型
- 默认假名链路首次运行时会下载并缓存 `JmdictFurigana / JmnedictFurigana`
- 视频烧录本身也比较耗时

第二次复用缓存会快很多。

如果你需要强制离线运行，可以先准备好自定义 furigana 资源，或者设置：

```bash
export NICOKARA_DISABLE_AUTO_FURIGANA_DOWNLOAD=1
```

## 12. 推荐使用流程

如果你是第一次做一首歌，推荐这样操作：

1. 先准备好 `mp4` 和干净的歌词 `txt`
2. 运行 `python3 build_nicokara_video.py "input.mp4" "lyrics.txt" --skip-burn`
3. 检查 `*.nicokara.json` 和 `*.karaoke.ass`
4. 运行 `nicokara-ruby-diagnostics *.nicokara.json --overrides-template ruby_overrides.json`
5. 如果歌词、读音或边界不满意，修改歌词或覆写文件后重跑
6. 最后去掉 `--skip-burn` 生成成片
